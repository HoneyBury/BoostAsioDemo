# v3.3.2 P5 Control Plane Gate

Date: 2026-05-17

## Scope

This pass tightens the default P5 control-plane gate without requiring Docker,
kind, or envtest assets on ordinary release runners.

## Changes

- `scripts/verify_control_plane_gate.py` now uses repository-local Go caches:
  - `runtime/go-cache/build`
  - `runtime/go-cache/mod`
- Added `scripts/check_operator_manifests.py` and wired it into the default P5 gate.
- The manifest gate validates:
  - `BoostGatewayCluster` CRD identity, status subresource, component spec fields, and status schema.
  - RBAC resources for CR/status, workloads, services, configmaps, secrets, cert-manager certificates, leases, and events.
  - manager Deployment service account, metrics port, probe port, readiness probe, and liveness probe.
  - sample `BoostGatewayCluster` contains all six components and gateway management port.
- Updated the current-state doc, reliability matrix, release checklist, fixed-runner playbook, and Operator README.
